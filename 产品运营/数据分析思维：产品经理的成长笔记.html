◆ 1.2 理解大脑并善用它

>> 压力有一个临界点，如果没有超过那个临界点，它是可以给人动力的，并没有什么危害。但是超过临界点，且长时间无法正确缓解压力，则会严重损伤人的认知，降低免疫系统，损伤学习能力。若这种状态长期存在，会让人产生无助的感觉，严重时会发展成抑郁症。

◆ 1.6 学习

>> 能成为顶尖高手并没有什么秘诀，而是对基本技能有更深的理解。

>> 学习的过程，就是在自己的大脑中建立相关技能的神经连接，将每一步的技能都内化成潜意识。任何领域的专家与新手的一个重要区别就是：专家不需用太多意识就能从细节中把握宏观的问题本质

>> 提出问题，尝试解决问题，探索解决方案并量化方案和获取反馈的过程。

>> “学海无涯苦作舟”这句话的前半部分是正确的，因为学习确实没有终点。但这句话后面的部分是错误的，学习不是苦的，它能勾起人类探索未知的兴趣，它的不确定性对人充满诱惑，这两种感觉组成了学习。

◆ 1.8 写作

>> 《自控力》一书中阐述道：人脑像一个求知欲很强的学生，对经验有着超乎大家想象的反应。如果你每天都让大脑学数学，它就会越来越擅长数学。如果你让它忧虑，它就会越来越忧虑。如果你让它专注，它就会越来越专注。你的大脑会根据你的要求重新塑型。就像通过锻炼能增加肌肉一样，通过一定的训练，大脑中某些区域的密度会变大，会聚集更多的灰质。

◆ 1.10 把压力当朋友

>> 学习新观点；练习、鼓励自己采纳和应用新思维；和别人分享该观点。用正面的角度去看待压力，把压力当朋友，只有这样我们才不会被压力压垮。

◆ 2.1 习惯

>> 任何一个好广告都要有亮点：第一点就是我能给你一个明确的暗示，第二点就是我给你清晰的奖励。

◆ 2.2 工作整理术

>> 成立一个团队，搞清楚什么是自己擅长的，坦承什么是自己不熟悉的。把自己不擅长的工作委派给别人，并且要信任他们，放手让别人去做。

◆ 2.4 游戏方法论

>> 游戏的方法论是有科学依据和参考价值的，它将刻意练习的3F原则： Focus（专注）、Feedback（反馈）与Fix it（纠正）展现得淋漓尽致。

◆ 2.8 怎样当个称职的教练

>> 自我责任。
  • 1．Goal：目标设定的常用问题你的目标是什么？如果你知道答案的话，那是什么？具体的目标是什么？什么时候实现？实现目标的标志是什么？如果需要量化的话，拿什么量化你的目标？
  • 2．Reality：关于现状的常用问题目前的状况怎样？你怎么知道这是准确的信息？这是什么时候发生的？这种情况发生的频率如何？你都做了些什么去实现目标？都有谁与此相关？他们对此分别是什么态度？是什么原因让你不能实现目标？你自身的原因有哪些？在目标不能实现的时候你有什么感觉？是什么令你……其他相关的因素有哪些？你都试着采取过哪些行动？
  • 3．Option：有哪些选择为改变目前的情况，你能做什么？可供选择的方法有哪些？你曾经见过或听说过别人有哪些做法？如果……会发生什么？哪一种选择你认为是最有可能成功的？这些选择的优缺点是什么？请陈述你觉得采取行动的可能性，打分。调整哪个指标，可以提高行动的可能性？
  • 4．Will：要做什么下一步是什么？何时是你采取下一步的最好时机？可能遇到的障碍是什么？你需要什么支持？谁可能对此有帮助？你何时需要支持，以及如何获得支持？

◆ 3.1 数据分析与产品设计

>> 设计产品不是以产品为主，而是以用户为主，而研究用户很多时候可以从心理学的角度来进行，将数据分析、心理学与产品的设计、营销结合起来，形成一种不断优化的机制，即可为用户提供一种更好的服务体验。

>> 心理学家欧文·比德曼估计“一个成年人一生可能要接触3万件不同的物品”，暂且不论确切的数目究竟是多少，单是想象就会明白这么多的物品无疑会使人们的日常生活复杂化。

>> 腾讯张志东的观点：在移动时代，如果你不够克制和敬畏，自然有比你克制、比你更用心的新产品出现，移动时代可供用户选择的产品大大增加了。

>> 据分析的场景多用在产品上线运行阶段。通过数据分析可以寻找到用户使用该产品的痛点，通过这些数据来优化产品的细节，这也是一个不断迭代的过程。

◆ 3.2 产品的功能很多，用的人有多少

>> 在移动设备上，过深的层级会使用户忽视该功能，所以并不是客户没有需求，只是客户不知道

>> 优秀的设计是设计人员和用户之间沟通良好的体现，只不过这种沟通要靠产品来体现。我个人觉得产品某项功能使用量的多少，很大程度上是由产品的设计人员所决定的。

◆ 3.3 设计心理学

>> 设计最重要的原则包括：可视性、限制因素、预设用途、自然匹配和反馈。

◆ 3.5 心理学、数据分析与产品营销的结合

>> 人类本性中有一些怪癖。其中一个怪癖就是：我们总是把注意力集中到自己会失去什么，而不是会得到什么上。

◆ 3.6 产品与数据创新

>> 马云说：“数据不在于挖掘，数据在于分享和交换，将两种完全不同的数据结合在一起，可能会突然得到一种新的产品或新的服务。”

◆ 3.7 提出好问题

>> 当遇到问题时，并不需要事必躬亲或亲自决策，而是可以采用向团队成员提问题的方式，让他们自己思考，互相分享彼此的观点和见解，从而共同寻求解决方案。

>> 好问题应当是有效的问题，因为问题可以分为两种：一种是有效问题，一种是无效问题。它们的判断标准是：这个问题是在赋予人能量（赋能）还是在消除人的能量（去能）。去能的无效问题会打击被问人的自信心和积极性；赋能的有效问题能让人产生积极心态，能够调动大家的积极性，进而更好地完成任务。提出好问题的原则是坦诚。如果不确定自己的问题是否合适，那就真诚地向对方坦白。

◆ 4.1 一次引发“失控”的数据分析

>> 数据的作用主要有三个，第一个是还原场景，第二个是记录反馈，第三个是量化人性。

>> 但其背后所牵扯到的利益却是真实存在的。在资源稀缺的前提下，数据分析的结果往往成为资源再分配的一个依据。而一旦出现资源的再分配，就必然会牵动既得利益群体的抗争。

◆ 4.2 幸存者偏差

>> “幸存者偏差”的案例说明：在数据分析的过程中，样本的选取不当有时候会误导分析的结论。

◆ 4.3 理性的无知

>> 理性的无知，不是真正的一无所知，而是从个人的利益出发，人们觉得自己保持无知状态是有益的，他们理性地选择了无知。

◆ 4.4 “数据造假”

>> 数据是要分析的，很多时候我们要从多个维度去审视所获得的信息，审视自己的认知是否存在缺陷。否则，很容易陷入别人为我们设下的陷阱。

◆ 4.5 猫和老鼠的数据攻防战

>> 所谓一流的才智，就是能够在心中拥有两种相互对立思维的同时，又能够保持正常判断的能力。一个人能够同时具备各为极端的两种思维理念，然后能根据实际状况的不同予以正确的区别和运用，这才是对立思维的完美统一。

>> 在做一件事情之前，先预测有什么因素会让这件事情最终走向失败，在把所有的因素都穷举出来后，再去做这件事，会比盲目地乐观更能达到最终目标。

>> 一个人单独思考时，总会有很多的盲区。当多个人一起思考时，可以看到一个人单独思考所看不到的盲点，做数据分析最需要的就是多角度思考，无论是正面的还是负面的，让更多不同角色的人参与非常重要。

◆ 4.6 数据分析中的陷阱与破解之道

>> 一些事实，对一般公司都有用：一是一个人的高考成绩和毕业院校跟他的工作能力并没有关系；二是一个人是否曾经在编程比赛中获奖，也和工作能力没关系；三是一个人在大学的学习成绩和工作表现有关系，但只在毕业后前三年内有效。

>> 破解陷阱之道

>> 数据是客观的，但是解读数据的人是主观的。

>> 不要过度依赖数据去寻找普适的标准，人与人之间的差别比我们想象的大得多，想要通过计算平均值来寻找一种标准是没有意义的。

>> 数据分析不是单纯地做数学题，不能为了分析而分析，而是应该围绕业务的问题展开。

◆ 4.7 当数据分析遇上数据安全和隐私

>> 数据安全和数据的隐私是一个相对的问题，企业不可能完全不让自己的职员接触到数据，同时企业也不可能让数据的流转失控。

◆ 4.10 数据分析的正确打开方式

>> 数据的作用总结为三个
  • 第一个作用：还原场景。
  • 第二个作用：记录反馈。
  • 第三个作用：量化人性。

>> 对失去某种东西的恐惧，似乎要比获得同一物品的渴望，更能激发人们的行动力。每当有东西获取起来比从前难，我们拥有它的自由受了限制，我们就越发想要得到它。

>> 我们需要问自己：做完这个数据分析后，别人看了结果后会产生什么样的反应？这些反应会引起什么反馈？这些反馈是不是我们做这次数据分析的初衷？

>> 那些显而易见的数据背后，有没有被忽视的“沉默样本”？在数据分析后我们提出的“理性”解决方案会不会造成“非理性”的结果？我们看到的数据是会真实地反映事物本质，还是别人用来迷惑我们的？

>> 只有正确地认识数据，才能正确地利用数据。在做数据分析时，对待数据我们必须要有一个求证的心态，并需要警惕那些被人处理过的二手数据

◆ 5.1 跨界这件事

>> 跨界一个最大的好处，就是可以跳出自己原有的领域去看问题，无论是个人还是企业。

◆ 5.2 我“不务正业”的故事

>> 掌握和精通每一项技能，都需要长时间的刻意练习，这样让技能变成潜意识

>> 自我暗示式地激励自己，不要自我批判

◆ 5.5 当数据遇上情感

>> 有些事情是“大数据”不擅长的：数据不懂社交。

>> 数据不懂背景

>> 大数据无法解决大问题。

>> 数据偏爱潮流，忽视杰作。

◆ 5.6 《点球成金》中的数据感悟

>> 既然选了，与其整天去后悔选择，不如找方法把事情做好。

>> 每个企业都有资源依赖症。但关于阻碍创新的根源，与其说是利益和制度，倒不如说是人自身的思维定势限制了人对未来的探索能力。

◆ 5.7 正确预测的副作用

>> “自证预言”（Self-fulfilling Prophecy），意思是指人会不自觉地按已知的预言来行事，最终令预言发生。

◆ 7.2 创新的窘境

>> 应对创新窘境的方法：第一个方法：收购其他的有不同流程、不同能力的公司。第二个方法：成立一个独立的机构来应对。

>> 产品在发展的过程中，不同的阶段会有不同的侧重点，一般可分四个阶段：第一个阶段看重功能性，例如硬盘的容量。第二个阶段看重可靠性，例如硬盘的稳定性。第三个阶段看重便携性，例如硬盘要小。第四个阶段看重价格。

◆ 7.7 面对变革的银行与员工

>> 不怕机器取代人，最怕人像机器一样思考。
